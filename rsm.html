Project Detail

Russel Street Medical — Clinic Website

A fully responsive multi-page clinic website for a Melbourne CBD medical practice, featuring an image carousel, a validated patient booking system, and a PHP-powered admin portal with session-based authentication and CSV data management.

‹ ›

Project Context

Russel Street Medical is a small clinic. The brief was to design and build their web presence from scratch: a public-facing site that introduces the practice, its staff, and its services, alongside a patient booking system and a secure internal portal for admin staff to manage appointment requests and user accounts.

The project was built in two phases. The first phase delivered the static front-end - responsive layout, brand colours, sticky navigation, staff profiles, and a JavaScript-validated booking form. The second phase introduced a PHP back-end: server-side form validation, CSV-based data persistence, session authentication, and a full admin dashboard - all hosted on RMIT's Coreteaching servers.

Frontend · PHP · Web Development

The site uses a modular PHP architecture - shared header, footer, and utility functions are extracted into include files, keeping each page lean. The booking form implements dual-layer validation: HTML5 patterns and custom JavaScript on the client side (including a checksum-based patient ID validator), plus full server-side re-validation in PHP to guard against submissions with JavaScript disabled. All booking data is persisted to a CSV file and read back.

Responsive Home Page
Sticky nav with smooth scrolling, About Us section with fee table, staff profiles, service area, and an auto-cycling image carousel with manual prev/next controls.
Patient Booking Form
Patient ID field with live uppercase conversion and checksum validation, date picker with no past dates, stylised pill-group time-slot checkboxes, appointment reason dropdown with contextual advice panel.
Server-side Validation
PHP re-validates all POST data independently of JavaScript. Errors are rendered inline next to the offending field with state preserved, and confirmed bookings are appended to a CSV with a timestamp.
Admin Portal
Session-based login reading credentials from a CSV user store, booking table with human-readable date formatting, staff registration with duplicate prevention, and failed login logging to a separate access-attempts file.
